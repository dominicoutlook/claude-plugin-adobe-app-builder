# Adobe Commerce AI Coding Developer Tooling

## Overview

Adobe provides an AI coding tooling suite that integrates with Claude Code (and other agents) via MCP servers and agent skills. It enables context-aware code generation, debugging, deployment, and documentation for Adobe Commerce App Builder extensions.

Supported agents: Cursor, Claude Code, GitHub Copilot, Windsurf, Gemini CLI, OpenAI Codex, Cline, and 40+ others.

Reference: https://experienceleague.adobe.com/en/docs/commerce/cloud-service/migration/migration-tools/coding-tools

## Prerequisites

- Node.js LTS version
- npm or yarn package manager
- Git for version control
- A supported coding agent (see Setup)

## Setup

```bash
# 1. Install Adobe I/O CLI globally
npm install -g @adobe/aio-cli

# 2. Install required plugins
aio plugins:install @adobe/aio-cli-plugin-commerce
aio plugins:install @adobe/aio-cli-plugin-runtime
aio plugins:install @adobe/aio-cli-plugin-app

# 3. Clone a starter kit (integration or checkout)

# 4. Run setup wizard
aio commerce extensibility tools-setup
```

Interactive wizard that configures: starter kit selection, coding agent, and package manager. After running, restart your coding agent to pick up the new MCP servers and skills.

After installation, enable the MCP server in your IDE. For Cursor: open MCP Settings and toggle the `commerce-extensibility` server on. Run `aio auth login` for full RAG functionality.

## MCP Servers

The setup provisions the following MCP servers:

| Server | Purpose |
|--------|---------|
| `commerce-extensibility` | Commerce-specific tools, RAG doc search, extension workflows |
| `developer` | Core development functionality |
| `app-builder` | App Builder-specific operations (deploy, dev, invoke) |

## MCP Tools

These tools are available when the MCP servers are connected:

| Tool | Description |
|------|-------------|
| `aio-app-deploy` | Deploy App Builder application or specific actions |
| `aio-app-dev` | Start local development server |
| `aio-dev-invoke` | Invoke a runtime action with parameters |

Total tools discovered: ~10 (additional tools available via the commerce-extensibility server).

### Tool Usage Examples

```bash
# Deploy all actions
aio-app-deploy

# Deploy a specific action
aio-app-deploy --actions my-action

# Start local dev
aio-app-dev

# Invoke an action
aio-dev-invoke my-action --parameters '{"key": "value"}'
```

## Agent Skills (Slash Commands)

These slash commands are available inside a project configured with `tools-setup`:

| Command | When to Use |
|---------|-------------|
| `/architect` | Planning integrations, selecting events, designing data flows, making architectural decisions for App Builder extensions |
| `/developer` | Generating code, updating config files, implementing runtime actions following App Builder patterns |
| `/devops-engineer` | Deploying applications, configuring environments, troubleshooting deployment, setting up CI/CD, resolving onboarding errors |
| `/product-manager` | Starting new projects, defining acceptance criteria, clarifying business objectives, creating `REQUIREMENTS.md` |
| `/technical-writer` | Writing README, user guides, API docs, changelogs, ensuring documentation completeness |
| `/tester` | Writing unit tests, integration tests, validating security, ensuring code quality and coverage |
| `/tutor` (experimental) | Learning App Builder concepts, understanding events, getting guidance on development patterns |
| `/search-commerce-docs "query"` | Searching Adobe Commerce documentation inline |
| `/feedback` | Sending feedback and sanitized logs to Adobe |

## Best Practices

Adobe recommends a **four-phase protocol**:
1. **Requirements analysis** — use `/product-manager` to define acceptance criteria
2. **Architectural planning** — use `/architect` to design data flows and select events
3. **Code generation** — use `/developer` to implement actions
4. **Documentation** — use `/technical-writer` to write README and API docs

Additional guidance:
- Use Plan mode when available in your coding agent
- Test incrementally after each feature
- Test locally (`aio-app-dev`) before deploying
- Deploy incrementally rather than full applications at once

## Troubleshooting

- **Skills/tools not appearing**: Restart Claude Code after running `tools-setup`
- **RAG search not working**: Run `aio auth login` first — RAG requires authentication
- **Auth issues**: Run `aio auth logout` then `aio auth login`
- **CLI commands appearing instead of MCP tools**: Explicitly request MCP tool usage
- **`/feedback` missing**: Update to latest version with `aio commerce extensibility tools-setup`
